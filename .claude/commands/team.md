`tmp/agent-status.md`（稼働台帳・ローカルのみ／gitignore）を読み、**直近の状況をすばやく**本人に提示する。重い集約（全エージェント定義の列挙や説明の要約）はしない。**読むだけ・ファイルは変更しない**。

> ⚡ **即時に台帳だけ見たいなら `! npm run -s team`**（シェル直叩き＝モデル不要で `/workflows` 並みに即返る）。このスラッシュコマンドは要約も付くがモデルのターンが走る。

## やること（最小）
1. `tmp/agent-status.md` を読む。無ければ「**台帳なし（未稼働）**」と表示して終わり。
2. **直近の行（新しい順で 8〜12 件程度）** を、台帳の列のまま表で見せる（開始 / エージェント / タスク / 状態 / 結果・備考）。`実行中` / `要承認` / `要判断` は強調。
3. いま `run_in_background:true` で動いているサブエージェントがあれば、その分だけ harness の追跡状態を1行補足する（無ければ触れない）。

## 出力
- 台帳の最新行の表。
- 末尾に **「実行中」と「本人の判断待ち（要承認/要判断）」だけ**を 1〜3 行で要約。どちらも無ければ「**稼働中なし・判断待ちなし**」。

即時性を最優先（長い前置き・全ロスター列挙・各 description の要約は不要）。詳しいロスターが要るときだけ `.claude/agents/` を見る。
